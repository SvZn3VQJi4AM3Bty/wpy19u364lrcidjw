Key Override

keymap.c の key_overrides による割り当て:

  • LAlt 押下中にスペース（LT(L_SYM, KC_SPC)）を押す → Alt+Tab
  • LShift 押下中に変換（LT(L_ADJ, KC_LNG1)）のタップを押す → F14

AIへキーマップ修正を指示する方法

次の方法のいずれかで指定することが推奨です。

  • スロット番号 … 画面上の #37 のように出ている番号で「レイヤー2のスロット37を KC_XXX に」と書く（keymap.c の LAYOUT 引数の並びと一致）。
  • レイヤー + 見た目位置 … 「L1 を表示した状態で、Q の位置」「一番下の列の矢印の左」など。
  • いまのラベル … 「BkTab になっているキーを ○○ に」。

複数キーは箇条書きにすると安全です。カスタムキー（enum custom_keycodes）を増やす場合は、その旨も書いてください。

メンテナンス

表示ラベルの揃え先は keymap_us_jis_label_mapping_template.md の US ラベル列（物理スロット＝keymap.cのLAYOUT順）です。keymap.c を変えたら、まず keymap_us_jis_label_mapping_template.md の US/JIS を直し、続けてこの HTML の LAYERS の keys（US 表記）に反映してください。スロット番号は HTML 内で整合します。JIS は toJisLabel() により US から表示変換（差分は JIS_LABEL_MAP 等。表示のみ・ファーム無関係）。layer_state_set_user の LED を変えたら、レイヤー概要表の「LED」列も合わせてください。
